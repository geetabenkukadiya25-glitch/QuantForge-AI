5. STRATEGY BUILDER'S "EXECUTION PIPELINE" -- ARCHITECTURE-ONLY CONFIRMATION
--------------------------------------------------------------------------------

  app.strategy_builder.models.ExecutionPipeline / ExecutionStep are pure
  DATA classes (Pydantic frozen models): a step_index, a component_name,
  a component_kind string ("indicator"/"detector"/"rule"), and a
  depends_on tuple. ExecutionPipeline.describe() produces a human-readable
  TEXT description of the computation order.

  Neither class contains, calls, or references:
    - any order-placement API
    - any broker connection
    - any live/demo account
    - any actual computation of the underlying rule conditions (rule
      `condition` strings are opaque text, never parsed/evaluated)

  This confirms the Execution Pipeline is exactly what Phase 8 specified:
  an architectural DESCRIPTION of computation order, not an executor.

2. RESOLUTION ENGINE VERIFICATION
--------------------------------------------------------------------------------

  resolve_components(context) classifies every SDL IndicatorSpec:
    - type registered in IndicatorRegistry only  -> IndicatorReference
    - type registered in SMCRegistry only        -> DetectorReference
    - type registered in BOTH                     -> "ambiguous" (error)
    - type registered in NEITHER                   -> "missing" (error)
  Every filters/entry_rules/exit_rules entry becomes a RuleReference,
  condition text carried through untouched (never interpreted/evaluated).

  Verified live this audit against moving_average_cross.yaml:
    Indicators resolved: fast_ma -> SMA, slow_ma -> SMA
    Rules resolved: golden_cross, death_cross (entry_rules),
                     opposite_cross (exit_rules)
    depends_on correctly captured for dependency-graph construction.
